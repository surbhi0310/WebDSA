Memory Allocation (C++)

Deep understanding of memory is mandatory for FAANG interviews

HeapStackInterview Favorite

1️⃣ What is Memory Allocation?

Memory allocation is the process of reserving memory space for variables and data structures during program execution. In C++, memory can be allocated either statically or dynamically.

2️⃣ C++ Memory Layout (Very Important)

  • Stack → Function calls, local variables
  • Heap → Dynamic memory allocation
  • Global / Static → Global & static variables
  • Code Segment → Program instructions

Interview Tip: Stack is faster, Heap is flexible.

3️⃣ Static vs Dynamic Allocation

StaticDynamic
Compile timeRun time
Stack memoryHeap memory
Fixed sizeResizable
Auto deallocationManual deallocation

4️⃣ Dynamic Allocation using new & delete

// Single variable
int *p = new int(10);

// Array
int *arr = new int[5];

// Deallocation
delete p;
delete[] arr;

✔ Allocated on heap ✔ Must be manually freed to avoid memory leak

6️⃣ Memory Leak

A memory leak occurs when allocated memory is not released.

int *p = new int(10);
// forgot delete

Result: Memory wasted until program ends.

7️⃣ Dangling Pointer

int *p = new int(5);
delete p;

// p is dangling

Fix: Set pointer to NULL after delete.

8️⃣ Best Practices (FAANG)

  • Always initialize pointers
  • Delete dynamically allocated memory
  • Use STL containers instead of raw pointers
  • Prefer smart pointers (modern C++)
